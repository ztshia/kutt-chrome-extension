# Kutt.it 链接管理器 - Chrome 插件



# 📋 插件概述

「Kutt.it 链接管理器」是一款基于 Chrome Manifest V3 开发的浏览器扩展，专为 Kutt.it 短链接服务设计。它将 Kutt.it 的核心功能集成到浏览器中，支持一键缩短当前标签页 URL、自定义短链接后缀、管理历史链接（复制/统计/删除）等功能，全程中文界面，操作简洁高效，大幅提升短链接创建与管理的效率。

**⚠️ 重要说明**：本插件为 Kutt.it 第三方辅助工具，非官方开发，仅用于便捷对接 Kutt.it 服务，所有短链接数据均由 Kutt.it 平台存储与管理。

# ✨ 核心功能

- **一键缩短当前页**：无需打开 Kutt.it 官网，直接对当前浏览的网页生成短链接

- **自定义后缀（Slug）**：支持手动输入个性化短链接后缀，替代随机字符，便于记忆与传播

- **历史链接管理**：展示最近 20 条短链接记录，支持快速复制、查看访问统计、删除无效链接

- **自动复制功能**：生成短链接后自动复制到剪贴板，无需额外操作

- **快捷键支持**：默认 `Ctrl+Shift+K`（Windows）/ `Command+Shift+K`（Mac）快速触发缩短功能

- **全中文界面**：从插件弹窗到设置页面，全程中文提示，降低使用门槛

- **安全轻量**：基于 Manifest V3 开发，权限最小化申请，本地仅存储必要配置（API 密钥等），无数据上传风险

# 📦 环境要求

1. **浏览器**：Google Chrome 88 及以上版本（支持 Manifest V3 的 Chromium 内核浏览器均可，如 Edge、Brave 等）

2. **依赖服务**：需拥有 Kutt.it 账号及 API 密钥（免费注册：[https://kutt.it/](https://kutt.it/)）

# 📝 使用教程

## 1. 首次使用配置（必做）

使用前需绑定你的 Kutt.it API 密钥，步骤如下：

1. 点击 Chrome 插件栏的「Kutt.it 链接管理器」图标，打开插件弹窗

2. 弹窗会提示「需要 API 密钥」，点击「打开设置」按钮进入配置页面

3. 获取 Kutt.it API 密钥：
        打开 Kutt.it 官网，登录你的账号

4. 进入个人设置页（右上角头像 → Settings）

5. 在左侧菜单找到「API」选项，点击「Generate New API Key」生成密钥

6. 复制生成的 API 密钥（注意：仅显示一次，建议立即保存）

7. 在插件设置页面填写：
        **API 密钥**：粘贴刚复制的 Kutt.it API 密钥

8. **基础 URL**：默认填写 `https://kutt.it`（若使用自建 Kutt 服务，填写对应域名）

9. 勾选需要的功能选项（自动复制、自动打开统计页等）

10. 点击「保存设置」按钮，配置立即生效，弹窗会自动刷新为功能页面

## 2. 核心功能使用

### 功能一：缩短当前标签页 URL

1. 打开需要缩短的网页（确保 URL 以 http/https 开头）

2. 点击插件图标，在弹窗的「自定义后缀」输入框中：
        填写自定义后缀（如 `my-blog-2025`），仅支持字母、数字、连字符

3. 若不填写，系统会自动生成随机后缀

4. 点击「缩短当前标签页」按钮，等待 1-2 秒

5. 操作成功后，弹窗会提示「短链接已创建并复制到剪贴板」，直接粘贴即可使用

### 功能二：使用快捷键缩短

在当前标签页中，按下快捷键：

- Windows/Linux：`Ctrl + Shift + K`

- Mac：`Command + Shift + K`

插件会直接使用默认配置（无自定义后缀）生成短链接，并自动复制到剪贴板，同时在浏览器顶部显示提示。

### 功能三：管理历史链接

插件弹窗下方会显示最近创建的 20 条短链接记录，每条记录支持三个操作：

- **复制**：点击「复制」按钮，短链接会复制到剪贴板，按钮临时显示「已复制！」

- **统计**：点击「统计」按钮，会在新标签页打开该短链接的 Kutt.it 访问统计页面（展示点击量、来源等数据）

- **删除**：点击「删除」按钮，会弹出确认框，确认后删除该短链接（删除后不可恢复，需谨慎操作）

点击弹窗的「⟳」图标，可手动刷新历史链接列表。

### 功能四：修改设置

若需修改 API 密钥、基础 URL 或功能选项，可：

1. 在插件弹窗点击右上角「⚙️」图标

2. 或在扩展管理页找到本插件，点击「详情」→「扩展程序选项」

3. 修改后点击「保存设置」即可

# 🔧 技术细节

## 1. 核心技术栈

- **Manifest 版本**：Manifest V3（Chrome 推荐标准，更安全、性能更优）

- **前端技术**：原生 HTML/CSS/JavaScript（无框架依赖，轻量高效）

- **数据存储**：Chrome `storage.sync`（配置同步，多设备共享）

- **API 交互**：基于 Fetch API 对接 Kutt.it V2 接口

## 2. 权限说明

插件仅申请必要权限，保障用户隐私与浏览器安全，各权限用途如下：

|权限名称|用途说明|
|---|---|
|storage|存储 API 密钥、基础 URL、用户偏好设置（如自动复制开关）|
|clipboardWrite|将生成的短链接复制到系统剪贴板|
|tabs|获取当前标签页的 URL 用于缩短，以及打开统计页面|
|activeTab|临时获取当前活跃标签页的权限，避免全标签页权限申请|
|scripting|辅助实现剪贴板复制功能（兼容旧版 Chrome）|
|contextMenus|（预留功能）支持右键菜单快速缩短链接|
|host_permissions|允许插件调用 Kutt.it 及自建 Kutt 服务的 API 接口|
## 3. API 对接说明

插件核心功能基于 Kutt.it V2 API 实现，关键接口如下：

|接口路径|请求方法|功能说明|核心参数|
|---|---|---|---|
|/api/v2/links|POST|创建短链接|target（目标URL）、customurl（自定义后缀）|
|/api/v2/links?limit=20|GET|获取历史链接列表（最近20条）|无|
|/api/v2/links/{id}|DELETE|删除指定短链接|id（短链接ID）|
所有请求均携带 `X-API-KEY` 请求头进行身份验证，确保接口调用安全。

# ❌ 常见问题与解决方案

## Q1：点击「缩短」按钮后提示「无法创建链接」？

可能原因及解决：

- API 密钥错误/过期：重新生成 Kutt.it API 密钥并更新插件配置

- 自定义后缀已被占用：更换其他后缀或不填写（使用随机后缀）

- 网络问题：检查网络连接，确保能正常访问 `https://kutt.it`

- Kutt.it 服务异常：访问 Kutt.it 官网确认服务是否正常

## Q2：插件弹窗不显示历史链接？

- 点击弹窗的「⟳」图标手动刷新

- 检查 API 密钥权限，确保拥有链接查询权限

- 若刚创建链接，等待 10 秒后刷新（Kutt.it 数据同步有延迟）

## Q3：快捷键无法使用？

1. 检查是否与其他插件快捷键冲突：Chrome 地址栏输入 `chrome://extensions/shortcuts`，查看并修改本插件快捷键

2. 确保插件已正确安装并启用

## Q4：删除链接后，Kutt.it 官网仍能看到？

插件删除操作实时同步到 Kutt.it 平台，若官网仍显示，可能是浏览器缓存导致，刷新 Kutt.it 页面即可。

## Q5：使用自建 Kutt 服务，如何配置？

在插件设置页面的「基础 URL」中填写自建 Kutt 服务的域名（如 `https://kutt.mydomain.com`），确保该服务支持 V2 API 接口。

# 📄 隐私政策

本插件高度重视用户隐私，具体隐私政策如下：

1. **数据收集**：仅在用户本地 Chrome 浏览器的 `storage.sync` 中存储 API 密钥、基础 URL 和功能偏好设置，不收集任何个人身份信息、浏览记录或链接数据。

2. **数据使用**：存储的 API 密钥仅用于对接 Kutt.it API 接口，实现短链接的创建、查询与删除功能，无其他用途。

3. **数据共享**：不将任何用户数据共享给第三方，所有数据仅在用户本地与 Kutt.it 平台之间传输。

4. **数据删除**：用户可通过插件设置页面的「清除所有数据」按钮删除本地存储的配置，或卸载插件时自动清除所有相关数据。
